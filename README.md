## 'top' 명렁어
### 1. 개요

> top - 실행 중인 시스템 상태를 **실시간으로** 모니터링하는 명령어
- CPU, 메모리 사용량, 실행 중인 프로세스 목록을 **실시간으로** 갱신해서 보여줌.

### 2. 주요 출력 항목
`top` 실행 시 상단부(요약 정보):
- `uptime` : 시스템이 켜져 있는 시간
- `Tasks` : 전체/실행/대기/중지/좀비 프로세스 수0
- `%Cpu(s)` : CPU 사용 비율 (us, sy, ni, id, wa, hi, si 등) 
- `MiB Mem` : 실제 메모리 사용량
- `MiB Swap` : 스왑 영역 사용량

`top` 실행 시 하단부(프로세스 목록):
- PID, USER, PR, NI, VIRT, RES, %CPU, %MEM, TIME+, COMMAND 등

### 3. 유용한 옵션
- `-b` : **batch 모드** -> 실시간 인터렉션 없이 출력만 하도록 함.
- `-p` : 특정 PID만 모니터링함.
- `-1` : CPU 코어별 사용량 표시
- `-k` : 선택한 PID에 신호 보내기
- `-r` : 프로세스 nice값(우선순위) 변경

## 'ps' 명령어
### 1. 개요

> ps - 현재 실행 중인 프로세스들의 상태를 스냅샷 형태로 출력하는 명령어
- 스크린샷을 생각하면 이해가 편할 것. 정지화면이다.
- 스크립트나 로그 확인을 할 때 유용하다.

### 2. 유용한 옵션
- `-ef` : System V 스타일의 전체 프로세스 목록
- `-u username` : 특정 사용자 프로세스만 보기.
- `-p` : 특정 PID만 출력하기.
- `-aux` : 모든 사용자/터미널의 프로세스를 포괄적으로 보기.

## 'jobs' 명령어
### 1. 개요

> jobs - 현재 쉘에서 백그라운드로 실행죽인 **잡(job)들의 상태를 출력**
- **쉘 빌트인(shell builtin) 이기 때문에 보통 별도의 실행 파일이 아니라 쉘 내부에 구현되어 있음
- 현재 쉘에서 백그라운드로 실행 중인 잡이 없으면 아무것도 출력하지 않는다.

### 2. 주요 출력 항목
```
[1]- Running      sleep 30 &
[2]+ Running      sleep 45 &
```
- [1], [2] : job의 번호
- '-' '+' : 기본 포그라운드 대상
- 'Running', 'Stopped' : 상태 표시

### 3. 유용한 명령어
- `fg %1` : 마지막/지정된 job을 포그라운드로 이동시킴
- `bg %1` : 마지막/지정된 job을 백그라운드로 재개시킴

## 'kill' 명령어
### 1. 개요

> 'kill' - 프로세스에 **시그널(signal)을 보내서 실행 상태를 제어하는 명령어. 주로 프로세스 종료(terminate)에 사용됨.
- 프로그램적으로는 '이 시그널을 보냄'이다.
- 어떤 시그널을 보내느냐에 따라 정상 종료/강제종료/일시중지/재개 등 다양한 명령이 가능해진다.

### 2. 시그널의 예시

- "kill -l"을 하면 사용가능한 모든 시그널 목록을 알 수 있다. 워낙 많기 때문에, 예시 몇개만 잠깐 알아보겠다.

| 번호 | 이름        | 의미/용도                       |
| -- | --------- | --------------------------- |
| 15 | `SIGTERM` | 기본 종료 요청. 정리할 시간 주고 종료.     |
| 9  | `SIGKILL` | 강제 종료. 커널이 즉시 죽임, 정리 시간 없음. |
| 2  | `SIGINT`  | Ctrl + C 에 해당. 인터럽트 요청.     |
| 19 | `SIGSTOP` | 프로세스 일시 중지.                 |
| 18 | `SIGCONT` | 중지된 프로세스 재개.                |
